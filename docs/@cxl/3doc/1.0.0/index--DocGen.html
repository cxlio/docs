The DocGen interface outlines the configuration structure for generating documentation. It specifies essential options like the output directory, TypeScript configuration, and optional features such as sitemap generation and markdown rendering.

baseHref?: string

Sets the base URL for markdown links within the generated documentation.

clean: boolean

Removes all existing files from the output directory before generating documentation.

customJsDocTags?: string[]

Allows declaration of custom JSDoc tags for documentation.

cxlExtensions?: boolean

Enables support for Coaxial UI extensions within the generated documentation.

debug: boolean

Enables debug mode to print detailed output during documentation generation.

demoScripts?: string[]

List of scripts (paths) to include in the generated documentation demo output.

demoStyles?: string
docsJson?: string

Path to a custom docs.json file for configuring documentation generation.

exclude?: string[]

List of modules (paths) to exclude from documentation generation.

exports?: string[]

Treats specific symbols (paths) as exported even if not explicitly marked.

extra?: Section[]

Additional sections to include in the documentation.

file?: string[]

Allows parsing of specific files instead of an entire project.

followReferences?: boolean

Includes documentation from symbols referenced in project references.

headHtml?: string

Path to a file containing custom HTML to be added to the <head> element of the generated page.

markdown?: boolean

Enables rendering of markdown syntax within symbol descriptions.

modulePackage: Package

The package information for the current project, typically sourced from package.json.

noHtml?: boolean
outputDir: string

Specifies the output directory where the generated documentation will be saved.

packageJson: string

Path to the package.json file. Defaults to ./package.json if not specified.

packageName?: string

Sets the title of the generated documentation. Defaults to the name property in package.json.

packageRoot: string

The root directory of the package for resolving paths.

repository?: string

URL of the source code repository (e.g., GitHub repository URL).

repositoryLink?: string

Link to the repository for the documentation site.

rootDir?: string

Overrides the default root directory used for resolving TypeScript project file names.

scripts?: string[]

List of additional scripts (paths) to include in the generated documentation HTML.

sitemap?: string

Generates a sitemap for the documentation using the provided value as the base URL.

spa: boolean

Enables Single Page Application (SPA) mode for documentation.

summary: boolean

Enables generation of a summary.json file.

tsconfig: string

Path to the tsconfig.json file used for TypeScript compilation.
